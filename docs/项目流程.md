# GainLab 项目流程

> 从想法到产品的完整路径。每一步有明确的输入、输出和判断标准。

---

## 流程总览

```
调研验证 → 方向确认 → 方案文档 → 展示页 → MVP工具 → 外部验证 → PRD → 开发
   ✅         ✅         ✅         ✅        ⬜         ⬜       ⬜     ⬜
```

---

## Step 1：市场调研验证 ✅ 完成

**输入**：初始想法（AI 聊天操控仪表板）
**做什么**：四市场（加密/美股/A股/外汇）调研，验证需求真伪
**产出**：
- 4 份调研报告（`memory/research-*.md`）
- 70+ 轮搜索，三份报告交叉验证
- AI Agent 交易评分：4.5-5/10
**判断**：原方向（AI 聊天）评分不够高 → 转向
**结论**：AI 聊天方向放弃，数据+可视化方向确认

---

## Step 2：方向确认 ✅ 完成

**输入**：调研结论 + OpenClaw 结合分析
**做什么**：
- 评估 GainLab 与 OpenClaw 的关系（互补 vs 竞争）
- Agent-native 可视化蓝海调研
- 缺陷和风险评估
**产出**：
- 新定位：衍生品数据可视化基础设施
- 三层架构：数据管道 → 可嵌入组件 → Agent 接口
- 差异化：Perp DEX 数据 + 可嵌入 + Agent 可调用
- 风险评估：6/10 可行，窗口期 6-12 个月
**判断**：方向可行但有风险 → 需要小成本验证 → 继续
**结论**：方向确认，进入方案细化

---

## Step 3：方案文档 ✅ 部分完成

**输入**：确认的方向 + 调研数据
**做什么**：
- [x] 方案概要 v1.0（`docs/方案概要.md`）——定位、架构、差异化、收费、路线图
- [x] Telegraph 版本，TG 内可查看
- [x] Apple Notes 存档
- [ ] ⚠️ 需要补充：具体技术选型、API 设计、任务拆解
**产出**：方案概要文档
**判断**：概要层面完成，细节在 PRD 阶段补
**状态**：概要完成，细节待补

## Step 4：展示页 ✅ 完成

**输入**：方案概要
**做什么**：
- [x] 交互式 HTML 页面（KLineChart + 热力图 + MCP 动画 + 定价）
- [x] 接入真实数据（Binance/OKX/Bybit/Hyperliquid API）
- [x] 部署到 GitHub Pages
**产出**：
- 公网链接：https://ashersun1207.github.io/gainlab/
- GitHub repo：Ashersun1207/gainlab（public）
**判断**：能看到真数据 ✅，但还是"展示页"不是"工具"
**状态**：完成，但不是最终产品形态

---

## Step 5：MVP 工具 ⬜ 进行中

**输入**：展示页 + 真数据 API
**做什么**：
- [ ] 做一个纯工具页面（去掉展示性内容，只留核心功能）
- [ ] Funding Rate 仪表板：4 交易所 × 主流币种，排序、筛选、异常高亮
- [ ] K 线图增强：多币种切换、指标选择
- [ ] 自己先用起来，替代 CoinGlass 看资金费率

**产出**：一个自己每天会打开的工具页面
**判断标准**：
- 🟢 自己每天看行情会用它（比 CoinGlass 方便）→ 说明有价值
- 🟡 偶尔看看但还是用 CoinGlass → 需要改进
- 🔴 做完自己都不想用 → 方向有问题

**时间**：1 周
**关键原则**：做"有用的工具"，不做"好看的展示"

---

## Step 6：外部验证 ⬜ 待做

**输入**：MVP 工具（自己验证可用后）
**做什么**：
- [ ] 发到 crypto TG 群（中文+英文）
- [ ] 发到 Crypto Twitter
- [ ] 发到 Reddit r/CryptoCurrency, r/BitcoinMarkets
- [ ] 观察：有没有人打开？停留多久？有没有人问问题？

**产出**：用户反馈数据
**判断标准**：
- 🟢 有人主动分享/收藏/提需求 → 需求验证通过，继续
- 🟡 有人看但没互动 → 可能是功能不够/定位不准，调整后再试
- 🔴 没人看 → 这个方向需求不成立，止损

**时间**：1-2 周观察期
**成本**：零（免费工具 + 免费发布渠道）

---

## Step 7：正式 PRD ⬜ 待做

**前置条件**：Step 6 验证通过（有人用、有人要）
**输入**：MVP 反馈 + 方案概要
**做什么**：
- [ ] 完整 PRD v2.0：系统架构、API 设计、数据模型、部署方案
- [ ] 技术选型确认：前端框架、后端框架、数据库、部署平台
- [ ] 任务拆解：每个功能拆成 1-3 天的开发任务
- [ ] 里程碑定义：每 2 周一个可交付版本

**产出**：PRD v2.0 + 任务列表
**判断**：文档足够细，AI 或人可以直接按文档开发

---

## Step 8：正式开发 ⬜ 待做

**前置条件**：PRD v2.0 完成
**按 PRD 路线图执行**：
- Phase 0（2周）：MCP Server + REST API
- Phase 1（4周）：3 个核心 Widget
- Phase 2（6周）：gainlab.ai 网站
- Phase 3（8周）：Perp DEX 数据管道
